Building an Agentic System for SWE-bench Lite

A Journey into Multi-Agent Collaboration

Introduction

When we set out to build Codart AI's system for the SWE-bench Lite challenge, we had a clear vision: create an intelligent system that could understand, analyze, and solve software engineering tasks with human-like reasoning. What emerged was a sophisticated multi-agent system that combines the power of large language models with specialized tools to tackle complex programming challenges.

The Multi-Agent Architecture

A Symphony of Specialized Intelligence

The Planner Agent

Our Strategic Thinker, powered by Claude 3.5 Sonnet. Analyzes GitHub issues and requirements, coordinates with the File Agent for code understanding, and creates detailed implementation plans.

The File Agent

Our Codebase Expert. Analyzes individual files in detail, provides code context and insights, and validates changes against the codebase.

The Implementation Planner

Our Change Coordinator. Acts as a bridge between planning and execution, ensuring changes match the plan exactly.

The Developer Agent

Our Code Implementer. Makes precise code changes, maintains code quality, and performs self-review.

The Collaboration Flow

  1. Planning Phase

    The Planner Agent receives a GitHub issue, analyzes requirements and context, coordinates with File Agent for code understanding, and creates a detailed implementation plan.

  2. Implementation Planning Phase

    The Implementation Planner receives the plan, tracks files that need modification, prepares for Developer Agent coordination, and ensures plan feasibility.

  3. Development Phase

    The Developer Agent receives specific changes, implements changes with proper validation, performs self-review and optimization, and maintains code quality.

  4. Validation Phase

    Changes are verified against the plan, code quality is checked, diffs are generated, and final validation occurs.
